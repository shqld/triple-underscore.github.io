  • ~UAは、上述の特色機能を可能にする~APIを~scriptに公開するべきである。 利用者は、~file~systemとのヤリトリに入ったときは，いつでも~transactionの取消／中止-を常に行えるような形で，~UIから通知を受ける。 利用者は、いかなる~file選択においても通知を受けられ，それを取消せる。 これらの~APIが、利用者の介在抜きに黙って呼出されることはない。 ◎ User agents should provide an API exposed to script that exposes the features above. The user is notified by UI anytime interaction with the file system takes place, giving the user full ability to cancel or abort the transaction. The user is notified of any file selections, and can cancel these. No invocations to these APIs occur silently without user intervention.
