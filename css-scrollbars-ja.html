CSS Scrollbars Module Level 1

要約

この仕様は、2000 年に~Windows~IE 5.5 により導入された，~scrollbarを色付ける能を標準~化する。 これは、既定の~platform~scrollbarの外観とごく異なる色~schemeを利用する~web~appを築くときに有用になる。 ◎ CSS Scrollbars standardizes the ability to color scrollbars introduced in 2000 by Windows IE 5.5. This is useful when building web applications which use color schemes very different from the appearance of default platform scrollbars.

1. 序論

注記： この仕様は、 W3C csswg-drafts GitHub repository にて css-scrollbars-1 として保守されている 。 ◎ Note: this specification is maintained in the W3C csswg-drafts GitHub repository as css-scrollbars-1 .

1.1. 視野

~INFORMATIVE

この~moduleが解決しようと意図する，~scrollbar周りの主な利用事例には、 文書~化された利用事例 に基づく，次の 3 つがある： ◎ Based on documented use-cases, there are three main use-cases around scrollbars this module intends to resolve:

  1. ~web~appの~UIにより適するように，~scrollbarを色付ける。 ◎ Color scrollbars to fit better into the UI of a web application.
  2. ~scrollしている区画が小さいときに，より細い~scrollbarを利用する。 ◎ Use thinner scrollbar when the scrolling area is small.
  3. ~scroll能に影響することなく，~custom化された~scrollbarを築く。 ◎ Build customized scrollbars without affecting scrollability.

これらの利用事例に取組むため、この~moduleは，要素に~scrollbarが在るときの色と幅を制御するための~propを追加する。 ◎ For addressing these use-cases, this module adds properties to control colors and width of scrollbars (if any) of an element.

~scrollbarの特定の部位を選択するための疑似要素は、この仕様の視野~外である。 ~WebKitによる~scrollbar用の疑似要素の実装は、誤って~webに公開された特色機能であるものと見なされる。 疑似要素による~approachに対する主な懸念は： ◎ Pseudo-elements for selecting specific parts of a scrollbar is out of scope. The WebKit implementation of pseudo-elements for scrollbar is considered to be a feature mistakenly exposed to the web. Main concerns against the pseudo-elements approach are:

  • ~OSは、より良い利用者~体験を供するため，自身による~scrollbarの設計を継続的に発展させる — 年月とともにこれを正確aに~model化するため，各種 疑似要素の組み合わせすべての能を超えるような。 ◎ Operating systems continuously evolve their scrollbar designs to provide better user experience, beyond the ability of any set of pseudo-elements to accurately model this over time.
  • ~scrollbarの構造は~platformごとに異なるので、相互運用能を試験するのは，より大変になる — 作者は，~engineに加えて~platformも織り込む必要が生じるので。 ◎ Different platforms have different scrollbar structure means testing interop is harder, because authors would need to take not only engine but also platforms into account.

2. ~scrollbarの色： `scrollbar-color^p ~prop

◎名 `scrollbar-color@p ◎値 `auto$v | `dark$v | `light$v | `color$vt{2} ◎初 `auto$v ◎適 `overflow$p が適用される~box ◎ boxes to which overflow applies ◎継 される ◎百 受容しない ◎算 指定された~keyword／ 2 個の算出された色 ◎ specified keyword or two computed colors ◎順 文法に従う ◎ア 算出された値~型による ◎表終

この~propは、要素の~scrollbarの色を設定することを，作者に許容する。 ◎ This property allows the author to set colors of an element’s scrollbars.

~UAは、根~要素に設定された `scrollbar-color^p 値を表示域に適用するモノトスル。 ◎ UAs must apply the scrollbar-color value set on the root element to the viewport.

注記： `overflow$p （および `overflow-*^p ）~propと違って、~HTML `body^e 要素~上に設定された `scrollbar-color^p 値は，表示域へは伝播されない。 ◎ Note: Unlike overflow (and overflow-*) properties, scrollbar-color value set on the HTML body element are not propagated to the viewport.

`auto@v
~scrollbar用の既定の~platform描画。 ◎ default platform rendering for the scrollbar.
`dark@v
暗い~scrollbarを示す — それは［ ~platformから供される~scrollbarの暗い変種, 暗い色を伴う`~custom~scrollbar$ ］のどちらにもなり得る。 ◎ show a dark scrollbar, which can be either a dark variant of scrollbar provided by the platform, or a custom scrollbar with dark colors.
【 `~custom~scrollbar@ が何を意味するのか，この仕様では特に言及されていない。 単に，［ ~platformから供されるものでも ~UAが既定で供するものでもない~scrollbar ］を意味すると見受けられる — それを超える解釈は、この仕様のみからは得られない。 】
`light@v
明るい~scrollbarを示す — それは［ ~platformから供される~scrollbarの明るい変種, 明るい色を伴う`~custom~scrollbar$ ］のどちらにもなり得る。 ◎ show a light scrollbar, which can be either a light variant of scrollbar provided by the platform, or a custom scrollbar with light colors.
`color$t{2}

1 個目の色を~scrollbarのツマミに適用し， 2 個目の色を~scrollbarの筋に適用する。 ◎ apply the first color to the thumb of the scrollbar, and the second color to the track of the scrollbar. ◎ Details:

  • 筋とは、~scrollbarの背景を指す — それは、一般に，~scrollしている位置に関わらず固定される。 ◎ Track refers to the background of the scrollbar, which is generally fixed regardless of the scrolling position.
  • ツマミとは、~scrollbar上の動かせる部位を指す — それは、通例的に，筋に沿って浮動する。 ◎ Thumb refers to the moving part of the scrollbar, which usually floats on top of the track.

この~propが `auto$v 以外の値に算出される場合、実装は，既定の~platform~UI描画より単純な~scrollbarを描画して，それに則って色付けてもヨイ。 ◎ If this property computes to value other than auto, implementations may render a simpler scrollbar than the default platform UI rendering, and color it accordingly.

各部に名前を付与した図式を追加する — http://www.howtocreate.co.uk/tutorials/scrlbar.html の様な。 ◎ (Note: add diagram showing the different named pieces - something like http://www.howtocreate.co.uk/tutorials/scrlbar.html)

頁の~styleに合致するような，色付けられた~scrollbarを伴う要素の例を追加する — 現時点でそれを~supportする~browserによる画像を伴わせて。 ◎ (Note: add example of an overflow element with colorized scrollbars to match page styling, PNG of the same in a browser that supports it currently)

実装は、下層の~platformに対応する部位が存在しない場合には，どの色も無視してヨイ。 ◎ Implementations may ignore any of the colors if the corresponding part do not exist on the underlying platform.

作者は、 `scrollbar-color^p ~propに特定の色~値を利用するときは、［ 指定した色どうしの~contrastが十分になる ］ことを確保するベキである。 ~UAは、~keyword値に対しては［ 自身が利用する色どうしの~contrastが十分になる ］ことを確保するベキである。 `WCAG20$r による技法 G183： ~contrast比 3:1 以上を利用して… 【日本語訳】 を見よ。 ◎ When using scrollbar-color property with specific color values, authors should ensure the specified colors have enough contrast between them. For keyword values, UAs should ensure the colors they use have enough contrast. See Techniques for WCAG 2.0: G183: Using a contrast ratio of 3:1 […] [WCAG20].

注記： 利用者が~scrollbarとヤリトリするとき（例： ~hoverする／作動化するなどにより）、実装は ~scrollbarを成すどの部位にどの~scrollbar色を適用するか改めてもヨイ。 ◎ Note: when a user interacts with a scrollbar (e.g. hovering or activating), implementations may alter which scrollbar colors apply to which scrollbar parts.

注記： ~IEは、各~scrollbar色~propの既定として有名~system色 【 `deprecated-system-color$t 】 を利用する。 関係する課題は `1956$issue を見よ。 ◎ Note: IE uses named System Colors as defaults for each of the scrollbar color properties. See related Issue 1956.

次の例は、~IEにおいて~scrollbar色を設定し直す（この例は https://www.w3.org/Style/Examples/007/scrollbars.en.html から導出されている）： ◎ The following example (derived from https://www.w3.org/Style/Examples/007/scrollbars.en.html) resets scrollbar colors in IE. 

html { 
    scrollbar-color: ThreeDFace Scrollbar;
}

3. ~scrollbarの太さ： `scrollbar-width^p ~prop

◎名 `scrollbar-width@p ◎値 `auto$v | `thin$v | `none$v | `length$vt ◎初 `auto$v ◎適 `overflow$p が適用される~box ◎ boxes to which overflow applies ◎継 されない ◎百 受容しない ◎算 指定された~keyword／絶対~長さ ◎ specified keyword or absolute length ◎順 文法に従う ◎ア 算出された値~型による ◎表終

この~propは、［ 要素の~scrollbarが示されるときの最大~太さを設定する ］ことを，作者に許容する。 ◎ This property allows the author to set the maximum thickness of an element’s scrollbars when they are shown.

`auto@v
実装は、既定の~platform~scrollbar幅を利用するモノトスル。 ◎ implementations must use the default platform scrollbar width.
`thin@v
実装は、適用-可能なときは， `auto$v より細い~scrollbarを利用するベキである。 これは、［ ~platformにより供される~scrollbarの細い変種, 既定の~platform~scrollbarより細い`~custom~scrollbar$ ］のどちらにもなり得る。 ◎ Implementations should use thinner scrollbars than auto when applicable. This may mean a thin variant of scrollbar provided by the platform, or a custom scrollbar thinner than the default platform scrollbar.
注記： 既定では，［ それ以上~細くできないか, 細くしても無為になるような，極小な~scrollbar ］しか備えない~platformもある。 その事例では、実装は，この値を `auto$v として扱ってもヨイ。 ◎ Note: Some platforms may only have a tiny scrollbar by default which cannot be or makes no sense to make thinner. In that case, implementations may treat this value as auto.
`none@v
実装は、~scrollbarを表示しないモノトスル — しかしながら，要素の~scroll能は影響されない。 【~scrollする手段が~scrollbarの他に無い環境の場合はどうなる？】 ◎ implementations must not display any scrollbar, however the element’s scrollability is not affected.
`length$t
~UAは、ある要素~用に 1 個~以上の~scrollbarを表示するときは、それらの~scrollbarの太さを `scrollbar-width$p に算出された長さ値~以下にするモノトスル。 負の `length$t 値は許容されない。 `scrollbar-width^p が 0 に算出される場合、要素の~scrollbarは隠される — しかしながら，その~scroll能は影響されない。 ◎ When a user agent displays one or more scrollbars for an element, the thickness of those scrollbars must be no more than the computed scrollbar-width length value. Negative <length> values are not allowed. If scrollbar-width computes to 0, the element’s scrollbars are hidden, however its scrollability is not affected.

注記： ~working-groupは、［ 長さ値を許容するか, ~keyword（ `auto^v, `thin^v, `none^v ）のみにするか, `scrollbar-width^p ~propをまるごと落とすか ］どうかについて，まだ合意に~~達してない。 （ `1958$issue ） ◎ Note: There is working group disagreement about whether to allow length values, or only have keywords (auto, thin, none), or drop the scrollbar-width property altogether. (Issue 1958)

~UAは、根~要素~上に設定された `scrollbar-width^p 値は，表示域に適用するモノトスル。 ◎ UAs must apply the scrollbar-width value set on the root element to the viewport.

注記： `overflow$p （および `overflow-*^p ）~propと違って、~HTML `body^e 要素~上に設定された `scrollbar-width^p 値は，表示域へは伝播されない。 ◎ Note: Unlike overflow (and overflow-*) properties, a scrollbar-width value set on the HTML body element is not propagated to the viewport.

注記： この仕様は、~scrollbarの正確な位置や形状, その~animation（ `fading or sliding in/out of view^en など）は，定義しない。 ◎ Note: This specification does not define the exact position or shape of the scrollbar, or any animation thereof, such as fading or sliding in/out of view.
