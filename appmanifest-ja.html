Web App Manifest

~ABSTRACT

この仕様は、~JSONに基づく形式の~fileを定義する — それは、~web~appに結付けられた~metadataを一極に集中する場を開発者に供する。 この~metadataは、［ ~web~appの名前, 各種~iconへの~link, および利用者が~web~appを立上げるときに開く~URLとして選好されるもの ］などを含む。 ~manifestは、［ ~app用の表示~mode（例：~fullscreen内）を設定するための能を供して，~web~app用の`既定の~screen方位$を宣言する ］ことも，開発者に許容する。 加えて，~web~appをある~URLに “~scopeする” ことを開発者に許容する。 これは、~manifestが適用される~URLを制約することに加え，他の~appから~web~app中へ “深く~linkする” 手段を供する。 ◎ This specification defines a JSON-based format file that provides developers with a centralized place to put metadata associated with a web application. This metadata includes, but is not limited to, the web application's name, links to icons, as well as the preferred URL to open when a user launches the web application. The manifest also allows developers to declare a default screen orientation for their web application, as well as providing the ability to set the display mode for the application (e.g., in fullscreen). 3.1. ~securityの考慮点
