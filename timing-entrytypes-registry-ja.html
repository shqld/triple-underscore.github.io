~ABSTRACT

この文書は、処理能~時列線 `PERFORMANCE-TIMELINE-2$r に利用される `PerformanceEntry.entryType$m 用の~registryを供する。 ◎ This document provides a registry of PerformanceEntry.entryType used in Performance Timeline [PERFORMANCE-TIMELINE-2].

1. 目的

~INFORMATIVE

処理能~時列線 `PERFORMANCE-TIMELINE-2$r は、［ ~web~appの全部的な~lifecycleから，様々な処理能~計量［ に~accessする／を~instrumentする／を検索取得する ］ことを~web開発者に可能化する ］ための~primitiveを定義する。 様々な処理能~dataの計量は， `PerformanceEntry$I を実装する~objにて~hostされ、その~interfaceの型は， `entryType$m で識別される。 ◎ Performance Timeline [PERFORMANCE-TIMELINE-2] defines primitives that enable web developers to access, instrument, and retrieve various performance metrics from the full lifecycle of a web application. The performance data of various metrics is hosted in PerformanceEntry objects. The type of interface for a PerformanceEntry object is identified with entryType.

この~registryの意図は、そのような~interface型を列挙するための，中心的な所在を供することにある。 ◎ This registry is intended to provide a central location for enumerating identified interface types of PerformanceEntry object.

2. 登録~entryの要件と更新-過程

各~entryには、次に挙げるもののうち［ “公な仕様” を含めるベキである／ 他のすべてを含めなければナラナイ ］：

`entryType^m 識別子 ◎ Each entry must include a unique identifier＼
`PerformanceEntry.entryType$m に利用される一意な識別子。 【！？ the scope of uniqueness is the entryType itself.】 ◎ used by PerformanceEntry.entryType, where the scope of uniqueness is the entryType itself.
公な仕様 ◎ Each entry should include a link that references a public available specification (PAS)＼
この~entry種別に結付けられている~interface型を定義している，公に可用な仕様を参照する~link。 【！PAS】 ◎ that defines the associated interface type.
要請者 ◎ Each entry must include contact information＼
要請者の連絡先~情報。 ◎ of the requestor.
`時列線から可用？$（ `availableFromTimeline^en ） ◎ Each entry must include an availableFromTimeline＼
この~entry種別は `Performance$I ~interfaceから可用になるかどうかを指示する真偽値。 ◎ boolean indicating whether this entry type is available from the Performance interface.
`最大~buffer~size$（ `maxBufferSize^en ） ◎ Each entry must include a maxBufferSize＼
この~entry種別~用の最大~buffer~sizeを指示する。 ◎ indicating the maximum buffer size for this entry type.

この~registryに対する更新は、識別子の［ 追加, 変更, 削除 ］に基づく。 どの個人も， `timing-entrytypes-registry^en ~repositoryへの~pull要請により，この~registryの更新を要請できる。 `Web Performance Working Group^en は、それを `upcoming meeting agenda^en に設置して，要請者に通知することになる。 要請の考慮と処置は、 `W3C Web Performance Working Group^en の同意による。 `the Chair^en は、外から来た要請者に通知し，それに則って~registryを更新することになる。 ◎ An update to this registry is an addition, change or deletion of an identifier. Any person can request an update to this registry by pull requests to the timing-entrytypes-registry repository. The Web Performance Working Group will place it on an upcoming meeting agenda and notify the requestor. Consideration and disposition of the request is by consensus of the W3C Web Performance Working Group. The Chair will then notify the requestor of the outcome and update the registry accordingly.
